# Daily Tasks for Servers

This guide outlines the standard daily tasks servers should perform when using MagiDesk POS.

---

## Opening Shift (Before Service Starts)

### 1. Log In and Verify System

**Time:** 2 minutes

**Tasks:**
- [ ] Log into MagiDesk POS
- [ ] Verify your name appears correctly
- [ ] Check all tables show as available (green)
- [ ] Verify printer is connected and has paper
- [ ] Print a test receipt to verify printer works

**If something is wrong:**
- Contact manager immediately
- Don't start service until issues are resolved

### 2. Review Menu and Specials

**Time:** 3 minutes

**Tasks:**
- [ ] Check today's menu is loaded
- [ ] Review any daily specials
- [ ] Note any items that are out of stock
- [ ] Check prices are correct
- [ ] Familiarize yourself with new items (if any)

**Why this matters:**
- You need to know what's available
- Customers will ask about specials
- You can't sell items that are out of stock

### 3. Check Your Assigned Tables

**Time:** 2 minutes

**Tasks:**
- [ ] Verify which tables you're assigned to (if applicable)
- [ ] Check tables are clean and ready
- [ ] Ensure table numbers are visible
- [ ] Check table status in system matches reality

---

## During Service (Active Serving)

### Task: Greeting and Seating Customers

**When customers arrive:**

1. **Greet customers** warmly
2. **Check table availability** in system
3. **Assign table** to customers
4. **Start table session** immediately:
   - Click on table
   - Click "Start Session"
   - Select your name
   - Click "Start"
5. **Provide menus** (if applicable)
6. **Take initial drink order** (if applicable)

**Important:**
- Always start session as soon as customers sit
- Timer starts automatically
- Don't forget this step!

### Task: Taking Orders

**When customers are ready to order:**

1. **Click on the table** in system
2. **Click "Add Items"**
3. **Listen carefully** to customer order
4. **Select items** from menu:
   - Browse categories
   - Or search for items
5. **Enter quantities** accurately
6. **Add modifiers** (size, toppings, etc.) if needed
7. **Repeat order back** to customer to confirm
8. **Click "Confirm Order"**
9. **Order is sent** to kitchen/bar

**Best Practices:**
- Write order down first (old school), then enter in system
- Double-check quantities
- Verify items are available before confirming
- Confirm order with customer before submitting

### Task: Checking Order Status

**While waiting for orders:**

1. **Click on table**
2. **Click "View Order"**
3. **Check status** of each item:
   - Pending (blue)
   - Preparing (yellow)
   - Ready (green)
4. **Coordinate with kitchen/bar** on timing
5. **Prepare to deliver** when items are ready

**Tips:**
- Check order status regularly
- Don't let items sit too long when ready
- Communicate with kitchen/bar

### Task: Delivering Orders

**When items are ready:**

1. **Pick up items** from kitchen/bar
2. **Verify items** match the order
3. **Deliver to table**
4. **Mark items as delivered** in system:
   - Click on table
   - Click "View Order"
   - Find the item
   - Click "Mark as Delivered"
5. **Check if customer needs anything else**

**Why mark as delivered:**
- Tracks what's been served
- Helps with accurate billing
- Shows kitchen/bar what's still pending

### Task: Processing Payments

**When customers are ready to pay:**

1. **Stop the table session:**
   - Click on table
   - Click "Stop Session"
   - Review the bill
2. **Present bill to customer:**
   - Show itemized breakdown
   - Explain time cost (if applicable)
   - Show total amount
3. **Collect payment:**
   - Ask how they'd like to pay
   - Process payment in system
4. **Process payment:**
   - Select payment method
   - Enter amount
   - Add tip (if applicable)
   - Apply discount (if manager approved)
   - Click "Process Payment"
5. **Print receipt:**
   - Receipt prints automatically
   - Hand receipt to customer
   - Thank customer
6. **Table becomes available** (green)

**Payment Best Practices:**
- Always verify bill total before processing
- Count cash carefully
- Verify card payments
- Always print receipt
- Thank customer for tip

### Task: Handling Special Requests

**Common scenarios:**

**Customer wants to add items:**
- Click "Add Items" on table
- Add new items
- Confirm order

**Customer wants to remove items:**
- Click "View Order"
- Remove item (if not yet prepared)
- Confirm removal

**Customer wants to split bill:**
- Use "Split Payment" feature
- Enter amounts for each payment method
- Verify total equals bill
- Process split payment

**Customer wants discount:**
- Check if you have authority
- If not, ask manager for approval
- Apply discount when approved
- Enter reason for discount

## Closing Shift (End of Service)

### 1. Process All Pending Payments

**Time:** 10-15 minutes

**Tasks:**
- [ ] Check all occupied tables
- [ ] Process any remaining payments
- [ ] Verify all bills are paid
- [ ] Stop all active sessions
- [ ] Ensure all tables are available (green)

**Important:**
- Don't leave with unpaid bills
- Don't leave active sessions running
- Verify all payments are processed

### 2. Review Your Sales

**Time:** 5 minutes

**Tasks:**
- [ ] View your sales for the shift
- [ ] Review number of tables served
- [ ] Check for any issues
- [ ] Note any problems to report

**Why this matters:**
- Helps you track your performance
- Identifies any issues
- Useful for reporting to manager

### 3. Final Checks

**Time:** 2 minutes

**Tasks:**
- [ ] Verify all tables are available (green)
- [ ] Check no active sessions remain
- [ ] Verify printer is off (if you're last to leave)
- [ ] Log out of system
- [ ] Report any issues to manager

## Throughout the Shift

### Regular Checks

**Every 30 minutes:**
- [ ] Check order statuses
- [ ] Verify tables are updated correctly
- [ ] Check for any system alerts
- [ ] Ensure printer has paper

**When busy:**
- [ ] Prioritize accuracy over speed
- [ ] Double-check orders before confirming
- [ ] Verify payments before processing
- [ ] Don't rush - mistakes cost time

**When slow:**
- [ ] Review menu and specials
- [ ] Check inventory (if you have access)
- [ ] Clean up any pending orders
- [ ] Prepare for next rush

## Common Daily Scenarios

### Scenario 1: Rush Hour

**What to do:**
- Stay calm and organized
- Take orders accurately (write down first)
- Enter orders in system as you go
- Check order status frequently
- Process payments efficiently
- Ask for help if overwhelmed

### Scenario 2: System is Slow

**What to do:**
- Be patient
- Don't click buttons multiple times
- Wait for system to respond
- If too slow, inform manager
- Use manual backup (paper) if needed

### Scenario 3: Printer Issues

**What to do:**
- Check printer is on and has paper
- Try printing test receipt
- If still not working, inform manager
- Can email receipt to customer (if enabled)
- Manager can print receipt later

### Scenario 4: Customer Dispute

**What to do:**
- Stay calm and professional
- Review bill with customer
- Check for errors
- If error found, correct it
- If no error, explain charges
- Get manager if needed
