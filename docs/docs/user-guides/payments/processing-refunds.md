# Processing Refunds

This guide explains how to process refunds for customers.

---

## When to Process Refunds

**Process refunds when:**
- Customer cancellation
- Wrong item delivered
- Customer dissatisfaction
- Item not delivered
- Manager-approved refunds
- Payment error correction

**Important:**
- Manager approval is usually required
- Refunds are tracked for audit
- Always enter a reason

---

## Step-by-Step Instructions

### Step 1: Find the Payment

**Go to Payments page:**

1. Click "Payments" in navigation menu
2. Find the payment to refund:
   - Search by table number
   - Search by date
   - Search by amount
   - Search by customer name (if available)

**Payment list shows:**
- Date and time
- Table number
- Server name
- Amount
- Payment method
- Status

### Step 2: Open Payment Details

**Click on the payment** you want to refund

**Payment details show:**
- Payment ID
- Bill details
- Payment amount
- Payment method
- Date and time
- Items paid for
- Refund history (if any)

### Step 3: Start Refund

**Click "Refund" button**

**Refund dialog appears:**
- Original payment amount
- Already refunded amount (if any)
- Remaining refundable amount
- Refund amount input
- Refund method selection
- Reason field

### Step 4: Enter Refund Amount

**Choose refund type:**

**Full Refund:**
- Enter full payment amount
- System may auto-fill this
- Refunds entire payment

**Partial Refund:**
- Enter partial amount
- Must be less than or equal to original
- Can refund multiple times (up to total)

**Example:**
- Original payment: $100.00
- Refund amount: $50.00 (partial)
- Remaining refundable: $50.00

### Step 5: Select Refund Method

**Choose how to refund:**

**Original Payment Method:**
- Refund to same method as payment
- Card payment → Refund to card
- Mobile payment → Refund to mobile

**Cash:**
- Refund in cash
- Even if original was card
- Requires manager approval

**Other Methods:**
- Wallet (if applicable)
- Store credit (if applicable)

### Step 6: Enter Reason

**Enter reason for refund:**
- "Customer cancellation"
- "Wrong item delivered"
- "Item not delivered"
- "Customer dissatisfaction"
- "Payment error"
- "Manager approved"

**Why this matters:**
- Required for audit
- Helps with reporting
- Tracks refund patterns
- Manager can review

### Step 7: Get Manager Approval

**Manager approval is usually required:**

1. **Request approval** from manager
2. **Explain situation** to manager
3. **Manager reviews:**
   - Is refund legitimate?
   - Is amount reasonable?
   - Does it follow policy?
4. **Manager approves or denies**
5. **If approved**, proceed with refund

### Step 8: Review Refund Summary

**Before processing, review:**

**Refund Details:**
- Original payment: $100.00
- Refund amount: $50.00
- Remaining refundable: $50.00
- Refund method: Cash
- Reason: Customer cancellation

**Verify:**
- ✅ Refund amount is correct
- ✅ Refund method is correct
- ✅ Reason is entered
- ✅ Manager approved (if required)

### Step 9: Process Refund

**Click "Process Refund"**

**System asks for confirmation:**
- "Are you sure you want to process this refund?"
- Shows refund details
- Final confirmation

**Click "Confirm" or "Yes"**

**What happens:**
- ✅ Refund is processed
- ✅ Refund is recorded
- ✅ Payment status updates
- ✅ Bill status updates (if applicable)
- ✅ Refund appears in history

## Refund Types

### Full Refund

**Refund entire payment:**
- Original: $100.00
- Refund: $100.00
- Remaining: $0.00

**When to use:**
- Complete cancellation
- Major issue
- Customer wants full refund

### Partial Refund

**Refund part of payment:**
- Original: $100.00
- Refund: $30.00
- Remaining: $70.00

**When to use:**
- One item was wrong
- Partial cancellation
- Specific amount refund

### Multiple Refunds

**Refund same payment multiple times:**
- Original: $100.00
- First refund: $50.00
- Second refund: $30.00
- Third refund: $20.00
- Total refunded: $100.00

**When to use:**
- Refunding in stages
- Different items at different times
- Up to original payment amount

## Important Notes

### ⚠️ Refund Rules

**Before processing:**
- ✅ Get manager approval (usually required)
- ✅ Verify refund is legitimate
- ✅ Check refund amount is valid
- ✅ Enter reason for refund
- ✅ Verify payment exists

**Validation:**
- Refund amount must be > 0
- Refund amount must be ≤ original payment
- Total refunds must be ≤ original payment
- Payment must exist

### ✅ Best Practices

- Always get manager approval
- Enter clear reasons
- Verify refund amount
- Check refund method
- Review refund summary
- Process promptly

### ❌ Common Mistakes

- Processing refund without approval
- Wrong refund amount
- Forgetting to enter reason
- Refunding more than original
- Not verifying payment details

## Special Situations

### Refunding Split Payment

**If original payment was split:**

- Can refund individual payment legs
- Or refund total amount
- Each refund is tracked separately
- Total refunded cannot exceed original

**Example:**
- Original: $50 card + $50 cash = $100
- Can refund: $30 card, $20 cash
- Or refund: Full $100 (both methods)

### Refunding with Discount

**If original payment had discount:**

- Refund is calculated on actual amount paid
- Discount is not refunded separately
- Refund amount = amount customer actually paid

### Refunding After Time

**If refund is requested later:**

- Can refund even if payment was days ago
- Check refund policy for time limits
- May need owner approval for old refunds

## Troubleshooting

### Problem: "Refund Amount Exceeds Payment"

**What it means:** Trying to refund more than was paid

**Solution:**
- Check original payment amount
- Check previous refunds (if any)
- Reduce refund amount
- Ensure total refunds ≤ original

### Problem: "Payment Not Found"

**What it means:** Payment doesn't exist or ID is wrong

**Solution:**
- Verify payment ID is correct
- Search by other criteria
- Check if payment was deleted
- Contact manager if needed

### Problem: "Cannot Process Refund"

**Possible causes:**
- System error
- Payment already fully refunded
- Manager approval missing

**Solutions:**
1. Check error message
2. Verify payment status
3. Get manager approval
4. Try again
5. Contact manager if issue persists
