# Daily Tasks for Managers

This guide outlines the standard daily tasks managers should perform when using MagiDesk POS.

---

## Opening Shift (Before Service)

### 1. System Check

**Time:** 5 minutes

**Tasks:**
- [ ] Log into system
- [ ] Verify manager dashboard loads
- [ ] Check all APIs are running (if you have access)
- [ ] Verify printer is working
- [ ] Test login for at least one server
- [ ] Check for any system alerts

**If issues found:**
- Contact IT support immediately
- Don't start service until resolved
- Have backup plan ready

### 2. Review Previous Day

**Time:** 10 minutes

**Tasks:**
- [ ] Review yesterday's sales report
- [ ] Check for any unresolved issues
- [ ] Review payment history
- [ ] Check for any refunds or discounts
- [ ] Review server activity
- [ ] Note any problems to address

**What to look for:**
- Unusual transactions
- System errors
- Server issues
- Customer complaints
- Inventory discrepancies

### 3. Inventory Check

**Time:** 10 minutes

**Tasks:**
- [ ] Review low stock alerts
- [ ] Check critical items (beer, popular food items)
- [ ] Verify stock levels match reality (spot check)
- [ ] Update stock if deliveries arrived
- [ ] Place orders for low stock items
- [ ] Note items that need attention

**Critical Items to Check:**
- Popular drinks
- Popular food items
- Items with low stock alerts
- Items that sell quickly

### 4. Menu Review

**Time:** 5 minutes

**Tasks:**
- [ ] Check menu is loaded correctly
- [ ] Verify prices are correct
- [ ] Check for items marked out of stock
- [ ] Review daily specials (if any)
- [ ] Ensure menu matches what's available

**Menu Issues to Fix:**
- Items that should be available but aren't
- Items that should be out of stock but aren't
- Incorrect prices
- Missing items

### 5. Staff Preparation

**Time:** 5 minutes

**Tasks:**
- [ ] Verify all servers can log in
- [ ] Check server assignments (if applicable)
- [ ] Review any new staff training needs
- [ ] Brief servers on any specials or changes
- [ ] Assign tables (if applicable)

## During Service (Active Management)

### Task: Monitor Operations

**Throughout the shift:**

- [ ] **Monitor active tables:**
  - Check table status
  - Verify sessions are started correctly
  - Ensure tables are stopped when customers leave

- [ ] **Monitor orders:**
  - Check order statuses
  - Ensure orders are being processed
  - Verify items are being delivered

- [ ] **Monitor payments:**
  - Check payment processing
  - Verify payments are being recorded
  - Watch for any payment issues

- [ ] **Support servers:**
  - Answer questions
  - Help with issues
  - Approve discounts when needed
  - Handle customer complaints

### Task: Approve Discounts

**When server requests discount:**

1. **Review request:**
   - Is discount legitimate?
   - Is amount reasonable?
   - Does it follow policy?

2. **If approved:**
   - Go to payment screen
   - Apply discount
   - Enter approval reason
   - Process payment

3. **If denied:**
   - Explain reason
   - Process payment without discount

**Discount Guidelines:**
- Set clear discount limits
- Require reasons for all discounts
- Track discount usage
- Review discounts regularly

### Task: Handle Customer Issues

**When customer has complaint:**

1. **Listen to customer** carefully
2. **Review the situation:**
   - Check order details
   - Review bill
   - Verify what happened

3. **Resolve the issue:**
   - Correct errors if found
   - Offer solutions (refund, discount, replacement)
   - Process refund if needed
   - Document the issue

4. **Follow up:**
   - Ensure customer is satisfied
   - Note issue for training
   - Update procedures if needed

### Task: Process Refunds

**When refund is needed:**

1. **Verify refund is legitimate**
2. **Review original payment**
3. **Process refund:**
   - Enter refund amount
   - Select refund method
   - Enter reason
   - Process refund

4. **Document refund:**
   - Note reason
   - Review with owner if large amount
   - Track for reporting

### Task: Update Inventory

**When deliveries arrive:**

1. **Receive delivery**
2. **Verify items received**
3. **Update stock in system:**
   - Find each item
   - Adjust stock level
   - Enter delivery as reason
   - Save changes

4. **Verify updates:**
   - Check stock levels updated
   - Verify items are available
   - Update menu if needed

## Emergency Tasks

### System Down

**If system goes down:**

1. **Assess the situation:**
   - How severe is the outage?
   - What's affected?
   - How long has it been down?

2. **Activate backup procedures:**
   - Use manual order taking
   - Accept cash payments only
   - Record everything for later entry

3. **Contact IT support:**
   - Report the issue
   - Provide details
   - Get ETA for resolution

4. **Keep business running:**
   - Don't panic
   - Use manual procedures
   - Keep customers informed
   - Document everything

### Payment Processing Down

**If payments can't be processed:**

1. **Accept cash only**
2. **Record card payments manually**
3. **Process payments later when system is back**
4. **Keep detailed records**
5. **Contact IT support**

### Inventory Issues

**If inventory is wrong:**

1. **Perform physical count**
2. **Compare with system**
3. **Adjust stock levels**
4. **Investigate discrepancies**
5. **Update procedures**
