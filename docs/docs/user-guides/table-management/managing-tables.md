# Managing Tables

This guide explains how to manage tables in MagiDesk POS.

---

## Understanding Table Status

### Table States

**Available (Green):**
- Table is free
- No customers
- Can start a session
- Ready for use

**Occupied (Red):**
- Table has active session
- Customers are there
- Timer is running
- Cannot start new session

**Needs Attention (Yellow):**
- Rare status
- System alert
- Check what's needed
- May need manager help

---

## Viewing Table Information

### Table Details

**When you click on a table, you see:**

**Available Table:**
- Table number
- Status: "Available"
- "Start Session" button
- Table type (Billiard/Bar)

**Occupied Table:**
- Table number
- Status: "Occupied"
- Server name
- Duration (how long customers have been there)
- Current order total
- "Stop Session" button
- "Add Items" button
- "View Order" button

## Common Table Operations

### Starting a Session

**When customers sit down:**
1. Click on available table (green)
2. Click "Start Session"
3. Select your name
4. Click "Start"

**See:** [Starting a Session](./starting-session.md) for details

### Stopping a Session

**When customers are ready to pay:**
1. Click on occupied table (red)
2. Click "Stop Session"
3. Review bill
4. Process payment

**See:** [Stopping a Session](./stopping-session.md) for details

### Adding Items

**To add items to a table:**
1. Click on occupied table
2. Click "Add Items"
3. Select items from menu
4. Confirm order

### Viewing Order

**To see current order:**
1. Click on occupied table
2. Click "View Order"
3. See all items
4. Check status of each item

## Table Management Tips

### Best Practices

‚úÖ **Do:**
- Start sessions immediately when customers sit
- Stop sessions when customers leave
- Keep tables updated accurately
- Check table status regularly
- Verify server name is correct

‚ùå **Don't:**
- Forget to start sessions
- Leave sessions running after customers leave
- Start multiple sessions on same table
- Ignore table status

### Common Scenarios

**Scenario 1: Customers Move Tables**
- Stop current session
- Start new session on new table
- Add existing order items to new table

**Scenario 2: Wrong Server Assigned**
- Stop the session
- Start again with correct server name
- Or ask manager to change it

**Scenario 3: Session Won't Start**
- Check if table is really available
- Check if another session exists
- Try refreshing page
- Contact manager if needed
